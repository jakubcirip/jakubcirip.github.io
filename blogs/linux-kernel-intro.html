Linux Kernel Intro

The Linux kernel is the core component of the Linux operating system. It acts as a bridge between hardware and software, managing system resources, memory, processes, and hardware communication.


In this introductory article, we'll look at what the kernel does, the difference between user space and kernel space, and how developers can begin exploring kernel development.

You'll also get tips on setting up a development environment and writing your first kernel module.


Basic Operating System (OS) concepts

  • Kernel

    Part of the OS that runs with elevated privileges.

  • User

    Applications run with lower privileges.

  • Kernel Mode

    A processor execution mode in which code can fully control the CPU (for example: deny/allow local CPU interrupts).

  • User Mode

    An execution mode in which the code has a certain limitation.
    An attempt at a privileged operation will generate an exception and the kernel will take control.

  • Kernel Space

    A memory area reserved for the kernel that is protected from direct access by user applications.
    Kernel mode code can directly access user space.

  • User Space

    A memory area reserved for a specific user process.

  • System Calls

    A set of APIs that applications expose. They represent the boundary where the execution mode switches from user mode to kernel mode.
    They are rarely changed in Linux to maintain application compatibility.

Kernel Architectures

  • Monolithic Kernel

    A kernel where there is no access protection between different kernel subsystems and where functions are called directly. Even monolithic kernels can be modular (e.g. support for runtime-loadable kernel modules).

  • Micro-kernel:

    Large parts of the kernel run as protected services in user space. The remaining code in kernel mode is smaller. It only implements the scheduler, the IPC (Inter Process Communication) mechanism, and basic memory management.

    • Advantages of Micro-kernel:

      Services are isolated, errors in one service will not affect others.

    • Disadvantages of Micro-kernel:

      Performance penalty because a simple function call in a monolithic kernel requires going through IPC and scheduling in microkernels.

  • Hybrid Kernels:

    A term (e.g. used for Windows, Mac OS X) that is often dismissed as marketing, because in them typical monolithic services still run in kernel mode.
